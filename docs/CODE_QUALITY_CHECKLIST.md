# 代码质量检查清单

**每次提交前必查**（强制执行）

## 📋 提交前必查清单

### 1. 代码风格匹配
- [ ] 阅读了项目3个同类文件
- [ ] 提取了风格特征（注释密度/docstring格式/命名习惯）
- [ ] 模仿了项目风格（不要"更好"，要"一样"）

### 2. docstring检查
- [ ] 类docstring ≤ 3行
- [ ] 函数docstring ≤ 2行
- [ ] 无列表化配置说明
- [ ] 无冗余解释基础概念

### 3. 注释检查
- [ ] 注释密度 < 20%
- [ ] 无显而易见的注释（如"初始化模型"）
- [ ] 只在复杂逻辑处注释

### 4. AI特征识别
- [ ] 代码是否过于"完美"？
- [ ] 注释是否多于必要？
- [ ] docstring是否像教科书？
- [ ] 是否缺乏人类的不完美？

**如果任何答案是"是" → 重写**

### 5. 自测验证
- [ ] 功能测试（至少3个场景）
- [ ] 边界测试（异常输入）
- [ ] 集成测试（与现有代码交互）

## 🚫 永久禁令

**以下行为绝对禁止**：

1. ❌ docstring > 5行
2. ❌ 配置项用列表枚举
3. ❌ 重复解释基础概念
4. ❌ 提交前未阅读项目代码
5. ❌ 缺乏自测证明

## 🔧 自动化检查

**使用代码质量检查脚本**：
```bash
./scripts/code_quality_check.sh <file>
```

**检查内容**：
- docstring行数
- 注释密度
- 列表化配置

## 💡 核心原则

1. **人类的不完美 > AI的完美**
2. **宁可不提交，不可交垃圾**
3. **一次做对比返工快100倍**
4. **风格匹配 > 功能实现**
