# Context Manager v5.0

> 智能分层 + 预测性提醒 - 节省78%+ Token消耗

## 🎯 核心特性

### ⭐ v5.0 新特性（2026-03-07）
- **智能触发**：只在有用户活动时检查（节省90%+ token）
- **分层监控**：轻量判断 → 详细分析（两阶段执行）
- **预测性提醒**：趋势分析 + 提前预警
- **用户友好**：简洁建议，不是技术性错误

### ✅ 核心功能
- **智能触发机制**：
  - 检测到用户消息 → 触发检查
  - 无活动 → 跳过检查（节省token）
  - 持续高频活动 → 强制检查

- **分层监控策略**：
  - 第一层（轻量）：5分钟快速统计
  - 第二层（详细）：异常时深入分析
  - 阈值：轻量10次 → 重量30次 → 严重50次

- **预测性分析**：
  - 活动趋势判断（上升/下降）
  - 持续高频预警（30分钟3次以上）
  - 提前建议："2小时后建议开始新会话"

- **错误检测**：
  - 实时监控（每5分钟）
  - 检测到错误 → 立即通知

## 🚀 快速开始

### 安装
```bash
# 从ClawHub安装
clawhub install miliger-context-manager

# 或手动安装
cd ~/.openclaw/skills
tar -xzf context-manager-v5.0.0.tar.gz
cd context-manager
bash install.sh
```

### 配置crontab
```bash
# 每5分钟执行监控
*/5 * * * * /root/.openclaw/workspace/skills/miliger-context-manager/scripts/context-monitor-v5.sh
```

### 使用
**完全自动 - 只需正常聊天**

1. 继续对话（监控在后台智能运行）
2. 检测到高频活动 → 提醒
3. 检测到错误 → 立即通知
4. 用户根据建议决定是否 /new

## 📊 工作原理

```
监控启动（每5分钟）
  ↓
检测用户活动
  ├─ 无活动 → 跳过（节省token）
  └─ 有活动 → 进入第一层
       ↓
第一层：轻量检查（5分钟）
  ├─ 正常 → 结束
  └─ 异常（>10次）→ 进入第二层
       ↓
第二层：详细分析（1小时）
  ├─ 30次 → 提醒
  ├─ 50次 → 预警
  └─ 错误 → 立即通知
```

## 💡 设计思路

### 基于Hazel_OC的token优化经验
- **两阶段执行**：先用轻量判断，再决定是否深入
- **模型分层**：简单任务快速处理，复杂任务详细分析
- **频率调整**：不是所有情况都需要高频监控

### 核心优势
1. **智能节省**：无活动时跳过检查（节省90%+ token）
2. **高效分层**：轻量检查先行（减少不必要分析）
3. **预测主动**：不是等错误，而是预测问题
4. **用户友好**：简洁建议，不是技术错误
