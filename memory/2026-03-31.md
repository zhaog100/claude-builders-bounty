## 💡 经验教训

### 1. 不同平台的付款流程不同
   - **RustChain**: PR → 验证 → **创建 Claim Issue** → 付款
   - **Algora/其他平台**: PR → 合并 → 自动付款
   - **教训**: 必须提前了解每个平台的付款流程

### 2. 付款地址类型必须匹配
   - 提交 PR 时要确认奖励类型(RTC vs USDT vs USD)
   - 提供正确类型的钱包地址
   - 不确定时提供多种地址(RTC + USDT + PayPal)

### 3. 及时创建 Claim Issue
   - **RustChain 要求**: PR 合并后立即创建 Claim Issue
   - **不要等待**: 主动创建 Claim Issue,不要等维护者主动联系
   - **超期提醒**: 超过 5 天未付款要主动提醒

### 4. PR 状态跟踪要及时更新
   - 定期检查 PR 状态(至少每天一次)
   - 及时更新 bounty-pr-tracker.json
   - 记录所有状态变化和跟进记录

### 5. ⚠️ 敏感数据脱敏
   - **API Key**: 永远不要在输出中显示完整的 API Key
   - **脱敏格式**: `sk-sp-****...****` (只显示前缀和后4位)
   - **建议用户**: 如果 API Key 已泄露,立即撤销并创建新的
   - **存储位置**: 敏感数据应存储在 `.env` 文件中,不要提交到 Git
