### 3. Bounty 付款可靠性评估方法
- **关键指标**：
  - 账号创建时间（老用户>新用户）
  - 历史付款记录（有>无）
  - Algora标签（有=可靠，无=高风险）
  - 仓库活跃度（star/fork/issue数量）
  - 竞争情况（认领人数）
- **风险评估等级**：
  - 低风险：Algora标签 + 历史付款 + 老用户
  - 中风险：老用户 + 无历史付款
  - 高风险：新用户 + 无Algora标签 + 无历史付款

### 4. Observability Stack 完整架构
- **核心组件**：
  - Prometheus：指标采集（15s间隔）
  - Grafana：可视化（5个预配置Dashboard）
  - Loki + Promtail：日志聚合（7天保留）
  - Tempo：分布式追踪（3天保留）
  - Alertmanager：告警路由（ntfy通知）
  - cAdvisor：容器指标
  - Node Exporter：主机指标
  - Uptime Kuma：服务可用性
  - Grafana OnCall：值班管理
- **数据保留策略**：
  - Prometheus：30天
  - Loki：7天
  - Tempo：3天
- **告警规则**：
  - 主机：CPU>80%、内存>90%、磁盘>85%、IO异常
  - 容器：重启>3次/小时、OOM、健康检查失败
  - 服务：5xx错误率>1%、响应时间P99>2s

2. **多账号**：用`&`符号分隔，每个账号的`pt_pin`和`pt_key`用`;`分隔

4. **got版本**：必须使用v11，v13不兼容
5. **定期维护**：Cookie 1-3个月过期，需定期更新
